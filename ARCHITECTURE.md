## Connection Flow

All peers connect to **MN hub** regardless of their IP subnet:

```
Client → vpn-server.example.com:51820 → MN Server → tunnel → SGP/other hubs
```

## Hub Subnet Mapping

| Hub | Subnet | Endpoint | Status |
|---|---|---|---|
| **MN** | `10.232.0.0/16` | `vpn-server.example.com:51820` | Active |
| **SGP** | `10.233.0.0/16` | `sgp-server.example.com:51820` | Active |
| **US** | `10.234.0.0/16` | `us-central.example.com:51820` | Future |
| **EU** | `10.235.0.0/16` | `eu-central.example.com:51820` | Future |

## Hub-to-Hub Routing

| Direction | Route | Via Peer |
|---|---|---|
| **MN → SGP** | `10.232.3.0/24`, `10.233.0.0/16` | `sgp_south` |
| **SGP → MN** | `10.232.0.0/16`, `10.233.0.0/16` | `mn_central` |

## Client Config Structure

```ini
[Interface]
PrivateKey = <auto-generated>
Address = <hub-subnet-ip>/32
DNS = <optional>

[Peer]
PublicKey = <mn-server-public-key>
Endpoint = vpn-server.example.com:51820
AllowedIPs = 10.232.0.0/16, 10.233.0.0/16
PersistentKeepalive = 25
```

- **Endpoint** — always MN (all traffic routes through MN hub)
- **AllowedIPs** — all active hub subnets (client can reach any peer in any hub)
- **Hub** flag controls only IP subnet allocation, not connection endpoint

## Directory Layout

```
/etc/wireguard/
├── wg0.conf              ← Interface config only (no peers)
├── wg-manage.conf        ← Hub/subnet/endpoint configuration
├── wg-manage.db          ← SQLite database (metadata, audit, IP pool)
├── peers/                ← Active peer .conf files
├── suspended/            ← Suspended peer .conf files
├── clients/              ← Generated client .conf files
├── private.key
└── public.key

/home/wg/                  ← Git repository
├── .version              ← Auto-incrementing version tracker
├── .gitignore
├── wg-manage.sh          ← Main script
├── PLAN.md               ← Implementation plan
├── ARCHITECTURE.md       ← This file
├── README.md
├── wg0.conf → /etc/wireguard/wg0.conf
├── peers/ → /etc/wireguard/peers/
├── suspended/ → /etc/wireguard/suspended/

/usr/local/bin/wg-manage → /home/wg/wg-manage.sh
```

## Database Schema

```
peers:        id, name, public_key, private_key, allowed_ips, hub, status, ...
ip_pool:      subnet_a, subnet_b, octet_c, octet_d, peer_id
audit_log:    timestamp, action, peer_name, ip, hub, version
config:       key, value
```

## Adding New Hubs

To add a new hub (e.g., US):

1. Edit `/etc/wireguard/wg-manage.conf`:
   ```bash
   WG_HUB_US_SUBNET="10.234.0.0/16"
   WG_HUB_US_ENDPOINT="us-central.example.com:51820"
   WG_CLIENT_ALLOWED_IPS="10.232.0.0/16 10.233.0.0/16 10.234.0.0/16"
   ```

2. Create hub-to-hub peer on MN:
   ```bash
   wg-manage add us_central --ip 10.234.0.1 --hub us
   # Add Endpoint to peer config manually
   ```

3. No code changes required — architecture supports unlimited hubs.
